IDEA-PLATFORM GAIA/CMIS 프로젝트

건설분야 종합사업관리 및 현장 사업관리를 위한 서비스 구축 프로젝트입니다.

프로젝트 개요

  • GAIA 및 CMIS 서비스 구축

    • GAIA : 이데아 플랫폼의 건설분야의 프로그램 레벨의 종합사업관리

    • CMIS: 이데아 플랫폼의 건설분야의 현장 사업관리

프로젝트 상세

  • 설계변경 또는 계약내용 변경 처리

  • 투입기술자, CM, 감리 등 인력관리

  • 기성검사 결과 처리, 물가변동 계약금액 조정

  • 관급자재 관리

  • 공정-공사비 관리, 공정부진시 만회대책 관리

  • 품질,안전 관리

  • 공사관리 행정 업무 처리 기능 및 승인 절차 처리

vscode

  • Shift+Ctrl+P 커맨트 폼에서 [ SVN: Checkout ]을 선택합니다.

  • svn url을 입력합니다.

  • 폴더를 입력합니다.

  • 체크아웃을 받으면 프로젝트가 생성됩니다.

프로젝트가 생성될 때 우측 하단에 [java 플러그인을 설치할까요?] 와 같이 플러그인을 추천하는데, 확장프로그램이 깔려있지 않으면 불편하므로 설치하면 됩니다. (물론 Extensions 탭에서 자유롭게 설치 삭제 가능합니다.)

  • 실행1. 왼쪽 자바프로젝트 패널에서 실행

  • 실행2. 디버그 탭에서 실행 (선택)

  • 실행3. main 함수가 있는 java 소스에서 실행 (선택)

java프로그램을 실행하는 방법은 여러가지가 있으며, gradle을 이용한 bootRun테스크를 사용해도 됩니다. 원하는 방법으로 실행하고 개발하세요.

Mybatis Mapper

  • namespace는 해당 패키지 + sqlx로 합니다.

  • 기타 작성은 mybatis 기본 작성규칙을 따릅니다.

  • 추가로 쿼리상단 주석을 `/* [{mapper-id}][{작성자}] */`로 필수로 남겨야 합니다.

    • mapper 샘플

<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="kr.co.ideait.gaiacmis.sample.sqlx">
    <select id="select" resultType="map">
        /* [kr.co.ideait.gaiacmis.sample.sqlx.select][hsjang]  */
        select cmn_cd_no, cmn_cd_nm_krn from sm_com_code
    </select>
</mapper>
쿼리 상단에 주석은 추후에 장애나 튜닝에 꼭필요한 부분으로 반드시 작성 부탁드립니다.

    • 기타 설정은 kr.co.ideait.core.config.MybatisConfig.java 파일 참조

  2. mapstruct (https://mapstruct.org/)

    • java object간 매핑을 위한 라이브러리로 코드를 대폭 줄일 수 있음

    • gradle build시 mapstruct annotation processor를 사용하여 코드를 생성함

    • 관련해서 IDE마다 해당 task에 대한 부분은 프로젝트 설정으로 추가해야하여, IDE별 설정을 참조하여 설정해야함

    • 컴파일 시 오류가 발생하면 해당 코드가 생성되지 않으니, 항상 컴파일 오류가 나지 않도록 해야함

  3. cache

    • ehcache3를 사용하여 캐시를 사용함

    • kr.co.ideait.core.config.CacheConfig.java 파일에서 캐시를 생성함

    • 캐시는 @Cacheable, @CacheEvict, @CachePut 어노테이션을 사용하여 처리

스프림 6.1부터 Spel사용과 관련한 제약사항이 있어 해당내용 패치등 확인이 필요함. https://github.com/spring-projects/spring-framework/wiki/Upgrading-to-Spring-Framework-6.x#parameter-name-retention

  1. jwt(https://jwt.io)

    • 인증 토큰을 위한 규젹은 jwt를 사용함

    • 프로젝트에서는 랜덤한 jwt키값을 서버 기동시에 사용하고 있으나, 필요에 따라 properties파일등에 고정된 값을 넣어서 사용할 수 있음.

  2. logback

    • 모든 로그는 logback을 사용하여 로깅하도록 함

    • lombok의 @Slf4j를 사용하여 로깅처리

try catch의 catch에서는 반드시 log.error(e.getMessage(), e)로 에러로깅 처리하여야 함

업무공통

인증/인가/로그인

  1. 인증

    • 인증에 대한 처리는 Spring Secutiry를 사용하여 처리함

    • 서버 기본 인증은 Cookie기반의 토큰이며, 이후 확장성을 고려하여 http Header에서도 토큰을 가져올 수 있도록 구성함

    • 인증 시간은 Cookie의 경우 쿠키 생성 시 시간을 확인하고, JWT토큰은 gaia.secret.jwt-expiration 설정값을 참고

    • kr.co.ideait.platform.gaiacairos.web.filter.AuthFilter.java 필터를 통하여 코어 인증 처리 함

    • 사용자 토큰 및 비지니스 정보를 가져와 인증정보를 생성하고, 메뉴 권한에 대한 처리를 담당함

    • 기타 내용은 해당 파일을 참고

  2. 인가

    • 권한 그룹 : 익명/ 사용자/ 관리자 (3등급)

    • 계약번호 [ADMIN(하드코딩)]에 설정된 권한 그룹을 가진 사용자는 관리자로 판단함

    • @IsUser @IsAdmin 어노테이션을 이용하여 인가 처리함

  3. 로그인 Overview

  • 임시로 포탈부분까지 구현되었으며, 추후 포탈 SSO 연동방식에 따라 변경될 수 있음

계약번호

  • 대부분의 업무는 프로젝트/계약번호에 의존성이 있으며 해당 정보는 로그인 시 선택되어 쿠키로 정보 저장함

  • 계약번호가 필요한 API에서는 계약번호가 없으면 에러가 발생하게 하는 어노테이션을 사용해야함(@RequiredProjectSelect)

  • 관리자는 특정 계약번호와 관련한 정보를 조회하거나, 수정할 수 있어야 해서, 해당 작업이 필요한 경우는 @RequiredProjectSelect(superChangeable=true)로 설정하고, parameter로 전달 받을 수 있게 처리함
